Analytics Builder aplicado ao admin Rails

Este bloco documenta como usar o conceito Explorer + Workspace + Inspector no sistema. A referência visual continua sendo este design system, mas a implementação Rails usa helpers e partials ax-* em vez de copiar o mock ou manter Bootstrap por baixo.

Contrato completo Componentes Playbook

Premissas

sem skin legado
  • Preservar Rails CRUD, params, permissões, submits, uploads e auditoria.
  • Extrair padrões para app/views/admin/shared/ui.
  • Remover Bootstrap quando o componente ax-* já cobre o comportamento.
  • Não atingir o site público ao ajustar tokens ou layout do admin.

Dashboard, listagens e cadastros devem usar a mesma régua lateral. Não criar padding local como 18px, .75rem ou 0 para compensar desalinhamento entre ax-contextbar, ax-main e ax-aside.

Aside direito

ax_aside_panel

A coluna direita é estruturalmente a mesma em todas as telas, mas o comportamento interno muda conforme o contexto. O shell mantém alinhamento, largura, colapso e separação entre ax-main e ax-aside.

Inspector de filtros

Usado em listagens. Contém campos, ações e grupos recolhíveis. Recolhido, não vira navegação por ícones.

Editor navegável

Usado no cadastro de imóveis. Quando recolhido, usa rail_body com ícones clicáveis para trocar de aba.

Painel informativo

Usado em configurações e previews. Header e body são dinâmicos, sem obrigação de navegação.

Botões

Use ax_button e ax_icon_button. Altura base: 32px; toolbar: 28px.

Quick modal

Cadastros rápidos usam ax_quick_modal, ax-quick-create, grid AX e aviso inline. Não usar modal fade ou alert Bootstrap.

Confirmação destrutiva

Remover documento, foto ou vínculo deve usar ax-confirm-submit. Evite window.confirm, alert e scripts inline.

Playbook de migração

ordem segura

1. Preserve comportamento

  • Rota, controller, strong params e permissões.
  • Submits, hidden fields, retorno e auditoria.
  • Uploads, TomSelect, Sortable e Stimulus existente.

2. Substitua apresentação

  • Troque row/col por ax_field_grid.
  • Troque alert por ax_inline_notice.
  • Troque modais Bootstrap por ax_quick_modal.

3. Valide e limpe

  • Uma aba/pane visível por vez.
  • Console sem erros no Atlas/in-app browser.
  • assets:precompile e zeitwerk:check quando aplicável.

Anti-padrões

não fazer
  • Não colocar ax-aside dentro de ax-main.
  • Não criar card dentro de card para resolver espaçamento.
  • Não usar legenda permanente quando tooltip resolve.
  • Não manter Bootstrap como base visual em área já migrada.
  • Não usar cor primária legada como default do novo layout.
  • Não criar controller específico de tela quando um ax-* genérico resolve.
  • Não usar código com vício de uma imobiliária específica.
  • Não afetar o site público ao ajustar o admin.
Ideia central: o usuário deve conseguir selecionar, configurar, validar e publicar sem trocar de tela. Explorer define contexto, Workspace mostra impacto, Inspector concentra propriedades e o painel inferior reduz incerteza com SQL, fórmulas, filtros e logs.
